# Unified Multi-Disease Retinal Disease Detection

A deep learning pipeline for multi-label retinal disease classification, trained on two heterogeneous fundus image datasets (**ODIR-5K** and **RFMiD**) unified under a single 8-class taxonomy, with cross-dataset generalization evaluation, confidence calibration, and Grad-CAM explainability.

## Overview

Most retinal disease classifiers are trained and evaluated on a single dataset, which limits real-world generalization. This project:

- Trains an **EfficientNetV2-S** backbone on ODIR-5K, then warm-starts and fine-tunes on a **combined ODIR-5K + RFMiD** dataset under a unified 8-class taxonomy
- Evaluates **cross-dataset generalization** — training on one dataset and testing on the other — to measure how well the model performs outside its original data distribution
- Applies **temperature scaling** for confidence calibration
- Uses **Grad-CAM** to visualize what the model attends to when making predictions
- Includes structured error analysis to identify systematic failure patterns

## Datasets

- **[ODIR-5K](https://www.kaggle.com/datasets/andrewmvd/ocular-disease-recognition-odir5k)** — Ocular Disease Intelligent Recognition dataset, ~6,392 training images across 8 disease labels
- **[RFMiD](https://www.kaggle.com/datasets/andrewmvd/retinal-disease-classification)** — Retinal Fundus Multi-disease Image Dataset, mapped into the same unified taxonomy

Datasets are downloaded automatically via `kagglehub` — no manual download or `kaggle.json` needed.

## Method

1. **Stage 1** — Train EfficientNetV2-S on ODIR-5K alone (frozen backbone warm-up, then full fine-tuning)
2. **Stage 2** — Warm-start from the Stage 1 checkpoint, fine-tune on the combined ODIR-5K + RFMiD dataset
3. **Evaluation** — Cross-dataset generalization matrix (train on A / test on B, for all combinations), per-class precision/recall/F1/AUC
4. **Calibration** — Temperature scaling fit on a held-out validation split
5. **Explainability** — Grad-CAM visualizations over the backbone's final convolutional layer
6. **Error Analysis** — Systematic false-positive/false-negative mining and failure case gallery

## Results

Cross-dataset generalization (macro F1):

| Train → Test | F1 | ROC-AUC |
|---|---|---|
| ODIR → ODIR | 0.529 | 0.910 |
| ODIR → RFMiD | 0.284 | 0.727 |
| RFMiD → RFMiD | 0.385 | 0.919 |
| RFMiD → ODIR | 0.276 | 0.670 |
| **Combined → ODIR** | **0.530** | **0.866** |
| **Combined → RFMiD** | **0.395** | **0.883** |

The model trained on the **combined** dataset generalizes better across both test sets than either single-source model — evidence that unifying heterogeneous datasets under one taxonomy improves robustness, not just in-distribution accuracy.

## Usage

This project is structured as a single Jupyter notebook (`retinal_disease_detection.ipynb`) organized into 15 sequential modules. Run cells **in order, top to bottom** — each module writes Python files that later modules import.

Designed to run on **Google Colab** (free-tier GPU) with Google Drive mounted for persistent checkpoint storage, so training can safely resume across sessions.

## Limitations

- Trained on a single free-tier GPU with a modest batch size and epoch budget — not at the scale of top published benchmarks
- Class imbalance (e.g. very few Hypertension examples) limits performance on rarer classes
- "Normal" is trained as an independent label rather than as the logical negation of all disease labels, so the model can (rarely) predict "Normal" alongside a disease label on the same image
- Temperature scaling calibration was only effective for 1 of 8 classes — a single global temperature may not suit this imbalanced multi-label setting well
